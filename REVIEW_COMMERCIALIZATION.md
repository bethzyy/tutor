# 商业化前审查报告

**审查日期**: 2026-04-12
**审查对象**: 个人成长导师 (Personal Growth Tutor) v1.0
**审查维度**: 心理专家 / 产品经理 / 软件架构师

---

## 一、项目现状概览

| 维度 | 现状 |
|------|------|
| 用户流程 | 目标设定 → 量表评估 → (可选)深度追问 → 计划生成 → 学习对话 → 步骤考核 → 最终考核 → 结业报告 |
| 心理量表 | 6 套标准化量表（CBF-PI-B, GSES, DASS-21, Growth Mindset, SGPS, ATMD） |
| 评估深度 | Stage 1: 标准化量表 + Stage 2: AI 3轮深度追问（CBT/MI/Big Five） |
| 技术栈 | React 18 + Vite + Tailwind / Express + SQLite(sql.js) / OpenAI-compatible API |
| 洞察系统 | 规则扫描（11条正则） + AI批量分析（每10条消息） + 镜像时刻 |

## 二、心理专家视角

### 2.1 科学性评估（优秀）

**强项：**
- 量表选用合理：CBF-PI-B、GSES、DASS-21 均有中文常模和已发表信效度数据
- `scoringEngine.js` 严格遵循心理测量学规范：反向计分、百分位计算（Abramowitz & Stegun 正态近似）、95% 置信区间
- 常模数据引用了具体样本量（N=430~1580），便于百分位解释
- 深度追问设计有理论基础（ABC模型、自动思维识别、改变标尺），3轮递进（情境→认知→核心信念）
- 规则扫描覆盖11种模式（绝对化语言、自我否定、非黑即白、读心术等），与 CBT 认知扭曲分类对齐

### 2.2 商业化必须改进的问题

| 优先级 | 问题 | 风险 | 建议 |
|--------|------|------|------|
| **P0** | **无知情同意机制** | 法律/伦理风险：未告知用户评估目的、数据用途、隐私保护 | 在首次使用前增加知情同意页面，明确说明数据仅用于个人成长建议，不构成临床诊断 |
| **P0** | **无危机干预机制** | DASS-21 可检测"偏高"压力，但无后续处理 | 当压力得分≥3.5或检测到"不如死"等高危表达时，自动展示心理援助热线（24小时）和就医建议 |
| **P0** | **免责声明缺失** | 用户可能误认为这是临床诊断 | 在报告页明确标注"本评估仅供参考，不构成医学诊断。如有心理健康困扰，请咨询专业心理师" |
| **P1** | **量表常模数据来源标注不够详细** | 可信度受质疑 | 每个量表的常模来源应标注具体文献（目前只有概括性引用，如"王孟成等(2011)"） |
| **P1** | **AI 生成的诊断问题未经验证** | AI生成的题目无信效度保障 | `diagnose_subject` 和 `diagnose_integrated` 模板让AI生成题目，应标注"非标准化评估"或完全改用固定题库 |
| **P1** | **深度追问无中止机制** | 可能触及用户心理创伤 | 允许用户随时跳过某问题，增加"这个问题让我不舒服"选项，触发温和退出 |
| **P2** | **缺乏重测追踪** | 无法验证干预效果 | 支持定期复评（如每月），对比前后百分位变化，展示成长轨迹 |
| **P2** | **无年龄段适配** | 未成年人使用有伦理问题 | 增加年龄筛查，16岁以下需家长同意；量表措辞需适配不同年龄段 |
| **P2** | **文化适应性** | 海外用户无法使用 | 量表需要英文/其他语言版本及对应常模 |

### 2.3 量表体系商业化建议

```
现有6量表 → 扩展方向：
├── 情绪维度: 新增 PHQ-9（抑郁筛查）、GAD-7（焦虑筛查）
├── 人际维度: 新增 IIP-32（人际问题量表）
├── 动机维度: 新增 SDT 自主动机分量表
├── 职业维度: 新增 CDSE（职业决策自我效能量表）
└── 复评追踪: 所有量表支持 T1→T2→T3 对比
```

## 三、产品经理视角

### 3.1 用户体验评估（良好）

**强项：**
- Notion 风格 UI 干净、一致，降低认知负荷
- 量表评估支持前后导航、自动推进、进度可视化
- 洞察系统的"镜像时刻"是差异化亮点
- 用户切换机制（多用户支持）设计完整

### 3.2 商业化必须改进的问题

| 优先级 | 问题 | 影响 | 建议 |
|--------|------|------|------|
| **P0** | **无认证系统** | 无法区分用户、无法收费、无法保护数据 | 接入手机号/微信登录，建立用户账户体系 |
| **P0** | **无支付系统** | 无法商业化 | 设计免费版/付费版分界：免费版2次评估+基础报告；付费版无限评估+深度画像+长期追踪 |
| **P0** | **无移动端适配** | 丢失 70% 流量 | PWA 优先，响应式布局优化，核心流程支持手机操作 |
| **P1** | **用户留存机制薄弱** | 一次性使用后无回访理由 | 增加每日推送（成长提醒）、每周进度报告、成就系统（连续打卡徽章） |
| **P1** | **社交分享功能缺失** | 无口碑传播渠道 | 结业证书支持图片导出+分享；成长里程碑支持朋友圈/小红书分享 |
| **P1** | **B端场景未覆盖** | 丢失心理咨询机构/企业EAP市场 | 增加咨询师端：查看客户评估报告、设置评估套餐、远程指导 |
| **P1** | **数据分析仪表盘缺失** | 用户无法看到长期成长趋势 | 增加"成长档案"页面：量表分数时间线、洞察变化、习惯改善曲线 |
| **P2** | **多语言支持** | 限制市场范围 | i18n 框架，首期支持中英双语 |
| **P2** | **内容运营缺失** | 无教程/引导/社区 | 增加成长知识库、用户社区、AI导师每日推送 |
| **P2** | **无 A/B 测试框架** | 产品优化缺乏数据支撑 | 接入实验平台，核心漏斗指标埋点 |

### 3.3 商业模式建议

```
收入模型:
├── C端订阅:
│   ├── 免费版: 2次基础评估/月，不含深度追问
│   ├── 标准版 ¥29/月: 无限评估 + 深度画像 + 成长追踪
│   └── 专业版 ¥99/月: 1对1 AI辅导 + 定制计划 + 专家报告
├── B端授权:
│   ├── 咨询师版: ¥199/月/咨询师，管理50个客户
│   └── 企业EAP: ¥999/月，100员工，含团队报告
└── 增值服务:
    ├── 专家解读报告 ¥49/次
    └── 定制领域模板 ¥99/领域
```

### 3.4 核心漏斗与指标

```
关键指标:
├── 注册 → 首次评估完成率 (目标 >70%)
├── 评估完成 → 计划生成率 (目标 >80%)
├── 计划生成 → 7日留存率 (目标 >40%)
├── 免费 → 付费转化率 (目标 >5%)
├── 月活用户 → 次月留存率 (目标 >60%)
└── NPS 净推荐值 (目标 >40)
```

## 四、软件架构师视角

### 4.1 架构评估（中等偏上）

**强项：**
- 模块化清晰：`services/` 分层（scoringEngine, assessmentService, insightEngine, ruleScanner）
- 量表与业务逻辑解耦：`scaleDefinitions.js` 纯数据定义，`scoringEngine.js` 纯函数
- AI 调用封装好：`callAIJson` 带重试和 JSON 提取
- Docker 部署支持
- 领域模板系统可扩展（`domainTemplates/` JSON 文件）

### 4.2 商业化必须改进的问题

| 优先级 | 问题 | 风险等级 | 建议 |
|--------|------|----------|------|
| **P0** | **内存会话状态丢失** | 高：服务器重启→用户评估进度丢失 | `assessmentService.js` 和 `deepAssessment.js` 的 `sessions` Map 必须持久化到 DB 或 Redis |
| **P0** | **无认证/鉴权** | 高：任何人可访问任何用户数据 | 接入 JWT + 用户认证，API 中间件校验 token |
| **P0** | **SQLite 并发瓶颈** | 高：sql.js 单进程同步写入，并发>50将严重阻塞 | 商业化必须迁移到 PostgreSQL 或至少 better-sqlite3（同步但更高效） |
| **P0** | **无输入校验/注入防护** | 高：用户输入直接拼入 AI prompt，存在 prompt injection 风险 | AI prompt 中的用户输入必须转义/截断；API 入参需要 Joi/Zod 校验 |
| **P1** | **内存限流器不跨实例** | 中：Docker 多实例时限流失效 | 改用 Redis-based rate limiter |
| **P1** | **无日志/监控体系** | 中：生产问题无法排查 | 接入结构化日志（pino/winston）+ 错误追踪（Sentry）+ 指标（Prometheus） |
| **P1** | **无测试** | 中：重构无安全网 | 至少覆盖 scoringEngine（纯函数易测）、API 端点集成测试、前端组件关键路径 |
| **P1** | **无数据加密** | 中：心理数据明文存储 | SQLite 文件加密或迁移到支持 TDE 的数据库；敏感字段 AES 加密 |
| **P2** | **前端无状态管理** | 低：当前规模可接受 | 引入 Zustand/Jotai，全局状态（用户信息、评估结果）不再依赖 prop drilling |
| **P2** | **无 API 版本化** | 低：后续迭代困难 | URL 前缀 `/api/v1/`，为破坏性变更预留空间 |
| **P2** | **Docker 配置不完善** | 低：缺少 nginx、SSL、健康监控 | 增加 nginx 反代 + Let's Encrypt + 自动备份脚本 |

### 4.3 推荐架构演进路线

```
Phase 1 (MVP → 商业化):
├── 认证系统: 手机号/微信 OAuth + JWT
├── 数据库迁移: SQLite → PostgreSQL (或 better-sqlite3 作为过渡)
├── 会话持久化: Map → Redis 或 DB 表
├── API 校验: Zod schema 验证所有入参
├── AI prompt 安全: 输入转义 + 长度限制
└── 基础监控: 结构化日志 + 错误告警

Phase 2 (规模化):
├── 微服务拆分: 评估服务 / AI服务 / 用户服务 / 支付服务
├── 消息队列: 评估报告异步生成（减少用户等待）
├── CDN + 静态资源分离
├── 自动化 CI/CD: GitHub Actions + 测试覆盖率门控
└── 数据备份: 每日快照 + 灾难恢复方案

Phase 3 (平台化):
├── B端 API: 机构接入 SDK
├── 多租户隔离
├── 实时协作: 咨询师-客户同屏
└── 合规认证: 等保二级 / SOC 2
```

## 五、综合评分

| 维度 | 当前评分 | 商业化门槛 | 差距 |
|------|----------|------------|------|
| **心理科学性** | 8.5/10 | 8.0 | 已达标，需补知情同意+危机干预 |
| **产品完整度** | 6.0/10 | 8.0 | 缺认证、支付、移动端、留存机制 |
| **技术架构** | 5.5/10 | 7.5 | 缺认证、持久化、测试、监控 |
| **安全合规** | 3.0/10 | 7.0 | 无加密、无审计、无隐私政策 |
| **商业准备度** | 2.0/10 | 6.0 | 无商业模式、无增长策略 |

**总体评估**: 产品在心理科学性上已经远超同类竞品，核心评估流程扎实。但商业化需要的认证、支付、安全合规、留存机制等基础设施几乎为零。建议按 Phase 分步推进，预计需要 2-3 个 sprint 完成 MVP 商业化改造。

## 六、优先级行动清单

### Sprint 1（核心安全 + 认证）
1. 增加用户认证系统（手机号 + 验证码）

3. 移动端响应式适配
4. 结业证书图片导出
5. 成长档案页面（历史追踪）

### Sprint 3（运营基础）
1. 日志/监控/错误追踪
2. 埋点 + 核心漏斗分析
3. 数据备份方案
4. 隐私政策 + 用户协议
5. 单元测试 + 集成测试（scoringEngine 优先）
