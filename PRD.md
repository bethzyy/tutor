## 1. 产品概述

### 1.1 产品定位

**一句话描述**: 基于标准化心理量表 + AI 深度访谈的科学成长伙伴。

**定位声明**: 个人成长导师是一款面向 18-40 岁自我提升需求人群的 AI 辅导产品。通过经过信效度验证的标准化心理量表评估用户的性格特质、自我效能、思维模式等核心维度，结合 AI 驱动的个性化深度追问和苏格拉底式对话，帮助用户发现自己的认知盲区和行为模式，制定科学可执行的成长计划，并在持续使用中追踪进步。

### 1.2 核心价值主张

| 对用户 | 对咨询师 | 对企业 |
|--------|----------|--------|
| 科学了解自己，而非星座式猜测 | AI辅助初筛，提高接诊效率 | 员工心理健康早筛+干预追踪 |
| AI 不评判，安全地探索内心 | 标准化量表节省测评时间 | 降低 EAP 成本 |
| 可量化成长，前后对比 | 远程追踪客户进展 | 数据驱动的团队心理报告 |

### 1.3 目标用户

**核心用户 (C端)**:
- 18-40 岁，城市白领/大学生
- 有自我提升需求但不知从何入手
- 对心理学感兴趣，愿意接受科学评估
- 付费意愿：¥20-100/月

**专业用户 (B端)**:
- 心理咨询师：用标准化量表快速建立客户画像
- 企业 HR/EAP 负责人：员工心理健康普查与追踪
- 教育机构：学生心理健康筛查

### 1.4 竞品分析

| 竞品 | 类型 | 优势 | 劣势 | 我们的差异化 |
|------|------|------|------|-------------|
| 心球/壹心理测评 | 测评工具 | 品牌大、内容多 | 单次测评无后续干预 | 评估→计划→辅导→追踪全闭环 |
| BetterHelp | 在线心理咨询 | 专业咨询师 | 贵($60-90/周) | AI辅助，价格1/10 |
| Forest/番茄TODO | 习惯养成 | 简单易用 | 无心理评估 | 科学评估驱动的习惯改善 |
| ChatGPT/Claude | AI对话 | 强大通用 | 无专业评估框架 | 标准化量表+AI对话结合 |

## 3. 功能需求详细说明

### 3.1 用户注册与认证 (新增)

**FR-AUTH-001 用户注册**
- 支持手机号+验证码注册
- 支持微信 OAuth 登录
- 注册时展示知情同意书，用户必须勾选同意
- 首次登录后引导设置昵称和目标

**FR-AUTH-002 用户登录**
- JWT token 认证，access token 30分钟 + refresh token 7天
- 支持记住登录状态
- 多设备登录限制（最多3设备）

**FR-AUTH-003 隐私保护**
- 心理评估数据标记为敏感数据
- 用户可导出/删除自己的所有数据（GDPR）
- 账号注销功能（30天冷静期）

### 3.2 知情同意与安全机制 (新增)

**FR-SAFE-001 知情同意**
- 首次使用前必须阅读并同意知情同意书
- 内容包括：评估目的、数据用途、隐私保护、不构成临床诊断
- 同意记录存入数据库（含时间戳）

**FR-SAFE-002 免责声明**
- 评估报告页固定显示免责声明
- 文案："本评估基于标准化心理量表，结果仅供参考，不构成医学诊断或临床建议。如果您正在经历心理困扰，请寻求专业心理健康服务。"

**FR-SAFE-003 危机干预**
- 规则扫描检测到高危表达（如自伤/自杀暗示）时：
  1. 立即在对话中展示关怀提示
  2. 展示24小时心理援助热线（全国：400-161-9995）
  3. 建议用户联系专业机构
  4. 后台记录事件（用于审计）
- DASS-21 压力子量表得分 ≥3.5 时：
  1. 报告页增加醒目提示框
  2. 推荐专业心理咨询资源
  3. 不使用"诊断"类措辞

**FR-SAFE-004 深度追问安全退出**
- 每轮追问提供"这个问题让我不舒服，跳过"选项
- 用户可随时中止深度追问，已有数据保留
- 中止后展示温和的退出语

### 3.3 评估系统 (核心增强)

**FR-ASSESS-001 标准化量表评估**
- 当前已有的6套量表保持不变
- 新增常模文献详细引用（具体论文标题+DOI）
- 量表信息页展示：量表名称、信度(Cronbach α)、题数、常模样本量、来源文献
- 量表评估流程保持当前设计：一次加载所有题目→前后导航→自动推进→完成后生成报告

**FR-ASSESS-002 评估报告增强**
- 报告新增内容：
  - 每个量表的常模对比图（用户分数 vs 常模均值 vs 常模+1SD）
  - 子维度详细分解（已有 `computeSubDimensions`，需在前端展示）
  - 置信区间可视化（已有 `confidenceInterval`，需在前端展示）
  - 首次评估 vs 历史评估对比（如有历史数据）
- 报告底部固定显示免责声明

**FR-ASSESS-003 深度追问优化**
- 3轮追问机制保持不变
- 新增：追问过程中的情绪检查（每轮结束后询问"还好吗？"）
- 新增：安全退出机制（FR-SAFE-004）
- 深度画像新增"建议干预方向"的可操作性描述

**FR-ASSESS-004 评估会话持久化**
- 所有评估会话状态存入数据库（不再使用内存 Map）
- 用户中断后可恢复评估进度
- 评估结果永久保存，支持历史查看

**FR-ASSESS-005 重测追踪 (新增)**
- 支持定期复评（用户自主触发或系统建议）
- 同一量表的前后对比：
  - 分数变化趋势线
  - 百分位变化
  - 显著性检验（配对t检验，N≥2时可用）
- 系统建议复评时机：完成计划后、30天后、用户主动请求

**FR-ASSESS-006 技能评估**
- 保持当前的领域模板机制
- 模板扩展：增加更多领域（考试准备、职场技能、语言学习等）
- Bloom认知层级展示在计划步骤中

### 3.4 成长计划与学习 (增强)

**FR-PLAN-001 计划生成**
- AI 生成计划保持当前逻辑
- 新增：计划生成时考虑深度画像的发现（如有）
- 新增：计划模板库（预设常见目标的最佳实践计划）

**FR-PLAN-002 学习对话**
- 苏格拉底式对话保持当前逻辑
- 新增：对话历史持久化 + 分页加载
- 新增：AI 回复引用洞察数据（如"结合你之前的评估，你在XX方面..."）

**FR-PLAN-003 考核系统**
- 步骤考核和最终考核保持当前逻辑
- 新增：考核失败时的具体改进建议
- 新增：考核通过后的鼓励反馈差异化

**FR-PLAN-004 结业证书增强**
- 证书增加：用户名、完成日期、各量表前后对比摘要
- 新增：证书图片导出（canvas截图，支持分享）
- 新增：证书分享链接（可选，有效期7天）

### 3.5 洞察系统 (增强)

**FR-INSIGHT-001 规则扫描**
- 保持当前11条规则
- 新增：规则库可在线更新（无需重启服务）
- 新增：置信度校准机制（基于用户确认/忽略反馈调整）

**FR-INSIGHT-002 AI批量分析**
- 保持当前每10条消息触发一次
- 新增：分析结果缓存，避免重复分析
- 新增：误报率统计（用户忽略比例过高时自动降低该类型置信度）

**FR-INSIGHT-003 镜像时刻**
- 保持当前逻辑
- 新增：镜像时刻频率控制（不过于频繁打扰用户）
- 新增：用户可以选择"再聊聊"后进入深度对话模式

### 3.6 成长档案 (新增)

**FR-PROFILE-001 档案首页**
- 展示用户的核心画像摘要：
  - 优势维度（量表高分项）
  - 待提升维度（量表低分项）
  - AI 发现的核心模式和信念
- 数据来源：最新评估报告 + 深度画像

**FR-PROFILE-002 时间线**
- 量表分数变化趋势图（折线图，每次评估一个数据点）
- 事件时间线：评估、计划开始、步骤完成、洞察确认等关键事件
- 支持按月/季度/全部筛选

**FR-PROFILE-003 成就系统**
- 徽章类型：
  - "首次评估" — 完成第一次综合评估
  - "自我探索者" — 完成深度追问
  - "行动派" — 完成第一个计划步骤
  - "坚持不懈" — 连续7天使用
  - "毕业" — 通过最终考核
  - "回访者" — 完成第二次评估
- 徽章展示在档案页，不可交易/不可隐藏
- 解锁时有动画效果

### 3.7 分享与社交 (新增)

**FR-SHARE-001 评估报告分享**
- 报告页增加"分享"按钮
- 生成精简版报告图片（隐藏详细分数，保留总体评价）
- 分享渠道：微信/朋友圈/保存图片

**FR-SHARE-002 结业证书分享**
- 证书页增加"保存图片"和"分享"按钮
- 证书图片包含：用户名、计划名称、完成日期、核心成就

**FR-SHARE-003 成就分享**
- 解锁新徽章时弹出分享引导
- 生成精美的成就卡片图片

### 3.8 B端功能 (新增)

**FR-BIZ-001 咨询师控制台**
- 咨询师注册需提供资质证明（审核机制）
- 客户管理：
  - 查看客户评估报告（需客户授权）
  - 设置评估套餐（选择推荐量表组合）
  - 添加私人笔记
- 远程指导：
  - 查看客户当前计划和进度
  - 给客户发送指导建议

**FR-BIZ-002 企业仪表盘**
- 管理员邀请员工使用
- 匿名统计仪表盘：
  - 团队整体压力水平分布
  - 各维度平均分
  - 高风险员工比例（匿名）
  - 干预效果追踪
- 数据不展示个人可识别信息

### 3.9 支付系统 (新增)

**FR-PAY-001 订阅计划**
- 免费版：每月2次基础评估，不含深度追问，不含成长档案
- 标准版（¥29/月）：无限评估 + 深度画像 + 成长追踪 + AI对话
- 专业版（¥99/月）：标准版 + 1对1 AI深度辅导 + 专家报告解读
- 年付享8折

**FR-PAY-002 支付渠道**
- 微信支付
- 支付宝
- Apple IAP（iOS端）

**FR-PAY-003 订阅管理**
- 用户可查看当前订阅状态
- 支持升级/降级/取消
- 取消后当前周期结束前仍可使用
- 续费提醒（到期前3天）

### 3.10 数据安全与合规 (新增)

**FR-SEC-001 数据加密**
- 数据库敏感字段 AES-256 加密（评估答案、洞察内容）
- 传输层 HTTPS 强制
- 数据库文件加密

**FR-SEC-002 数据导出**
- 用户可导出自己所有数据（JSON格式）
- 包含：个人信息、评估记录、对话历史、洞察记录
- 导出链接有效期24小时

**FR-SEC-003 数据删除**
- 用户可申请删除所有数据
- 30天冷静期后执行
- 删除前确认邮件

**FR-SEC-004 审计日志**
- 记录所有数据访问操作（谁、何时、访问了什么）
- 危机干预事件单独记录
- 日志保留2年

## 6. 数据模型

### 6.1 新增/修改表

```sql
-- 用户认证表
CREATE TABLE user_accounts (
    id INTEGER PRIMARY KEY,
    phone TEXT UNIQUE,
    wechat_openid TEXT UNIQUE,
    password_hash TEXT,           -- 仅手机注册
    nickname TEXT NOT NULL,
    avatar_url TEXT,
    role TEXT DEFAULT 'user',     -- user | counselor | admin
    status TEXT DEFAULT 'active', -- active | suspended | deleted
    created_at DATETIME DEFAULT (datetime('now')),
    last_login_at DATETIME,
    consent_given_at DATETIME,    -- 知情同意时间
    consent_version TEXT          -- 同意的版本号
);

-- 订阅表
CREATE TABLE subscriptions (
    id INTEGER PRIMARY KEY,
    user_id INTEGER NOT NULL,
    plan TEXT NOT NULL,            -- free | standard | professional
    status TEXT DEFAULT 'active',  -- active | cancelled | expired
    started_at DATETIME,
    expires_at DATETIME,
    cancelled_at DATETIME,
    payment_method TEXT,           -- wechat | alipay | apple
    FOREIGN KEY (user_id) REFERENCES user_accounts(id)
);

-- 危机干预日志
CREATE TABLE crisis_events (
    id INTEGER PRIMARY KEY,
    user_id INTEGER NOT NULL,
    trigger_type TEXT NOT NULL,    -- dass21_high | keyword_detection | ai_flag
    trigger_detail TEXT,
    response_type TEXT,            -- hotline_shown | resource_recommended
    created_at DATETIME DEFAULT (datetime('now'))
);

-- 成就表
CREATE TABLE achievements (
    id INTEGER PRIMARY KEY,
    user_id INTEGER NOT NULL,
    badge_type TEXT NOT NULL,
    unlocked_at DATETIME DEFAULT (datetime('now')),
    UNIQUE(user_id, badge_type)
);

-- 审计日志
CREATE TABLE audit_logs (
    id INTEGER PRIMARY KEY,
    user_id INTEGER,
    action TEXT NOT NULL,
    resource_type TEXT,
    resource_id INTEGER,
    ip_address TEXT,
    detail TEXT,
    created_at DATETIME DEFAULT (datetime('now'))
);
```

## 10. 风险与缓解

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| 心理数据泄露 | 低 | 极高 | 加密存储+访问控制+审计日志+渗透测试 |

| 用户增长慢 | 中 | 中 | SEO+内容营销+社交分享+免费增值 |

## 附录 B: 术语表

| 术语 | 定义 |
|------|------|
| 标准化量表 | 经过信效度验证的固定题目的心理测量工具 |
| 常模 | 大样本的参考数据，用于将个人分数与群体比较 |
| 百分位 | 个人分数在群体中的相对位置（如P75表示超过75%的人） |
| Cronbach α | 量表内部一致性信度指标，>0.7为可接受 |
| 反向计分 | 反向表述的题目，选高分实际得分低（用于避免默认偏差） |
